# AGENTS.md

このドキュメントは、このプロジェクトの**唯一の仕様書（Single Source of Truth）**です。

## 1. 目的
- 営業用ホームページとして、問い合わせ獲得につながる構成とデザインを提供する。
- 「最適な形を提案するホームページ制作」という価値を、信頼感ある見た目と導線で伝える。

## 2. 対象プロジェクト
- ルート: `AYHP/`
- 主な実装ファイル: `index.html`, `style.css`, `script.js`
- 補助ページ: `works.html`, `contact.html`

## 3. デザイン方針（固定）
- 清潔感、信頼感、やや上品。
- シンプルだが弱くない。
- 営業LPとして反応が取れる見た目を優先。
- 読みやすさ重視（余白、可読性、視線導線）。
- スマホ表示を同等以上に重視する。
- 過剰な装飾・過剰なアニメーションは避ける。

## 4. 実装方針（固定）
- 静的実装（HTML / CSS / JavaScript）を基本とする。
- 既存構成を活かし、必要最小限のHTML変更で改善する。
- 文言の大幅変更は行わない（依頼がある場合を除く）。

## 5. 変更管理ルール（最重要）

### 5.1 仕様優先の原則
- **AGENTS.md を唯一の正**とする。
- AGENTS.mdに未記載の内容は、原則実装しない。
- 実装内容が仕様と矛盾する場合、コードではなく先に仕様を更新する。

### 5.2 変更手順（必須）
変更は必ず次の順で行う。
1. まず AGENTS.md を確認する。
2. 変更したい箇所に対応する仕様項目を AGENTS.md で更新する。
3. 仕様更新後に実装を開始する。
4. 実装完了後、仕様とコードの一致確認を行う。
5. 最後に、変更箇所をすべて報告する。

### 5.3 禁止事項
- 仕様更新なしでの先行実装。
- 仕様にない独自判断の機能追加。
- 変更箇所の報告漏れ。
- スマホ表示確認を省略すること。

## 6. 実装前チェックリスト（毎回必須）
- [ ] AGENTS.mdの該当仕様を確認した
- [ ] 今回の変更内容をAGENTS.mdに反映した
- [ ] 変更対象ファイルを特定した

## 7. 実装後チェックリスト（毎回必須）
- [ ] 実装内容がAGENTS.mdと一致している
- [ ] 影響範囲（PC/スマホ）を確認した
- [ ] 不要な変更が混入していない
- [ ] 変更箇所を一覧化した

## 8. 変更報告フォーマット（毎回固定）
変更完了時は、必ず以下形式で報告する。

1. 仕様変更
- 変更した AGENTS.md の項目名
- 変更内容

2. 実装変更
- ファイル名
- 変更内容

3. 一致確認
- 仕様とコードが一致しているか
- 未反映・要確認事項（あれば）

## 9. 現在の固定仕様（要約）
- ヘッダー、ファーストビュー、CTA、理由、実績、流れ、料金、最終CTAを主軸にする。
- CTAは視認性を高くし、行動導線を明確にする。
- セクション間にメリハリをつけるが、色は統一感を維持する。
- カードUIは一貫した余白・角丸・境界・影ルールで設計する。
- スマホでは可読性とタップしやすさを優先する。
- 「選ばれる5つの理由」は PC表示で円形レイアウトに配置し、視覚上の表示順は 01 → 02 → 03 → 04 → 05 とする。
- 5枚のカード配置は `index.html` の DOM順（01〜05）を維持し、CSSで位置のみ制御する。
- 「自分の作りたいをつくる」セクションの PC / スマートフォンモックは、PC表示で左右に隣接配置し、詰めすぎない適度な間隔を確保する。
- 「自分の作りたいをつくる」セクションの PC / スマートフォンモック間隔は PC表示で 2cm を基準とする。
- ファーストビューは、画面中央から左半分に紺色オーバーレイを敷き、下方向に向かって薄くなるフェードを適用する（下の画像側の既存フェード仕様は変更しない）。
- ファーストビュー題名は左寄せで紺色オーバーレイ内に収め、2行表示を固定する。
- ファーストビューの紺色オーバーレイは右辺を斜めにし、下部境界は線が見えない自然なフェードとする。
- ファーストビューの紺色オーバーレイの斜め角度は、必要に応じてさらに強める。
- 紺色オーバーレイのフェードは「こんなお悩み相談ありませんか？」付近で境界が見えないよう、下端より手前で十分に透明化させる。
- 紺色オーバーレイの上部濃度は必要に応じて強め、同時に「こんなお悩み相談ありませんか？」付近の境界線が視認されないようフェードを最優先で調整する。
- ファーストビューの紺色オーバーレイは、上〜中段（無料で相談する付近）では濃度変化を抑え、下側で急に薄くなるフェード特性を基本とする。
- ファーストビューの紺色オーバーレイは、上部CTA（「制作実績を見る」）付近で終わる高さを基本とする。
- ファーストビューの紺色オーバーレイ（本体色＋グラデーション）は、上部CTA（「無料で相談する」）付近までで終える。
- ファーストビュー題名ブロックは、ヒーロー左側へ寄せた配置を維持する。
- ファーストビュー題名（「最適な形を提案するホームページ制作」）の縦位置は、必要に応じて下方向へ微調整可能とする。
- ファーストビュー題名は、依頼に応じて下方向へ追加調整を行える。
- ファーストビュー題名は、依頼時に `1cm` 単位で下方向調整を行える。
- ファーストビュー題名は、依頼に応じて `2cm` 程度まで下方向調整を許可する。
- ファーストビューのアイブロウ文言（例:「デザイン性 x 成果重視」）は、題名位置調整時に同時に下方向へ揃えて調整する。
- 「こんなお悩み相談ありませんか？」ブロックは、ヒーロー内で中央配置とする。
- 「こんなお悩み相談ありませんか？」ブロックは、直前CTAとの間隔を必要に応じて詰めて調整する。
- ファーストビュー内の箇条書き（メタ情報）と2つのCTAボタンは中央配置とする。
- ファーストビュー上部CTA（「無料で相談する」「制作実績を見る」）は、必要に応じて縦2段表示・横幅拡張・右方向への微調整を許可する。
- ファーストビュー上部CTAは、依頼に応じて左右位置を微調整し、ホバー時に浮き出る視覚効果を付与できる。
- ファーストビューでは、題名とその下の説明〜CTAブロックの間隔を十分に確保し、視認上の階層を明確にする。
- ファーストビューの題名下ブロック（説明〜CTA開始位置）は、指定がある場合 `3cm` を基準に下方向へオフセットする。
- ファーストビューの紺色オーバーレイの斜め形状は、必要に応じてより急角度へ調整する。
- ファーストビューのメタ情報は「オンライン相談対応 / 公開後サポート対応 / スマホ最適化標準」の3項目を基本とし、`/` 区切り・枠なし表示とする。
- ファーストビューのメタ3項目およびヒーロー内ボタン群は、依頼に応じて左方向へ微調整できる。
- ファーストビューの小文（ヒアリング文〜メタ3項目）は、紺色オーバーレイ範囲内に収まることを優先し、左寄せ調整を許可する。
- ファーストビューのリード文（「ヒアリングから設計〜作ります。」）は、PC表示で3行を目安にし、紺色オーバーレイ内に収める。
- ファーストビューのリード文は、PC表示で4行化を避け、3行を優先する。
- ファーストビューのリード文は、上部CTAの主ボタン（「無料で相談する」）位置を基準に配置を揃える。
- 「自分の作りたいをつくる」セクションのPCモックは、現状より左に寄せ、モック内部は中央揃えとする。
